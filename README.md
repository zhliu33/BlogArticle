# 章仓库说明

本仓库用于统一存放个人/团队所有博客文章的原始稿件、修订版本及相关资源，旨在实现博客内容的规范化管理、便捷归档与高效复用，同时为内容迭代、备份与分享提供可靠支撑。

## 仓库定位

作为博客内容的核心托管载体，兼顾「归档备份」与「创作管理」双重需求，既保障所有文章数据的安全存储，也方便创作者快速检索、修改稿件，同时为后续博客平台发布、内容二次传播提供基础支持。

## 内容范围

本仓库收录的内容包括但不限于：

- 博客文章原始稿件（以Markdown格式为主，兼容少量HTML格式）；

- 文章修订版本（按修改时间/版本号分类，保留修改痕迹，便于回溯）；

- 文章配套资源（如插入图片、封面图、引用文件等，统一存放于指定资源目录）；

- 文章元信息文档（记录每篇文章的发布平台、发布时间、标签、阅读量等基础信息）；

- 创作规范与模板（统一文章排版、标题格式、引用规范等，保障内容一致性）。

## 管理规范

1. 文件命名规范：文章文件统一以「日期-文章标题」命名（示例：20260315-如何高效管理博客仓库.md），避免特殊字符，确保检索便捷；

2. 版本管理：修订文章时，保留原始版本，新修订版本在文件名后添加版本号（示例：20260315-如何高效管理博客仓库-v2.md），并在文章末尾标注修改记录；

3. 资源管理：所有文章配套资源统一放入resources目录，按文章标题建立子目录分类存放，避免资源混乱；

4. 元信息更新：每发布一篇新文章或修改文章核心信息后，及时更新meta目录下的元信息文档，确保信息同步；

5. 权限管理：个人仓库建议开启私有权限，团队仓库可根据角色分配读写权限，避免误操作或内容泄露。

## 使用说明

- 新增文章：在articles对应目录下新建Markdown文件，可参考template目录下的文章模板撰写，配套资源放入resources对应子目录；

- 修改文章：复制原始文章文件，修改后按版本号命名，保留原始文件，同时更新元信息；

- 检索文章：可通过仓库搜索功能，按文章标题、日期、关键词检索，也可通过目录分类快速定位；

- 备份与同步：定期提交仓库更新，确保本地与远程仓库同步，避免内容丢失；如需跨平台发布，可直接从仓库复制稿件进行编辑发布。
